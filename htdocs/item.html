Запрет проведения заказа и реализации при превышении фиксированной суммы по договору. УТ 11.4, 11.5, ERP 2.4-2.5, КА 2.4-2.5

Расширение решает давнюю проблему - сделать так, чтобы больше фиксированной суммы договора не проводился заказ и реализация, при приеме заказа сразу чтобы видно было, запретить клиенту перебирать сумму договора.

Описание

Данное расширение работает на типовых конфигурация УТ, ERP, КА2.

Условия для работы:

  1. Порядок расчетов у договора "По расчетным документам"
  2. Флажок "Сумма договора фиксирована"
  3. Сумма больше 0 = потолок превышения

Плюсы от приобретение данного решения:

  1. Не проводит заказ клиента партнёрам, превысившим лимит кредита по договору.
  2. Не проводит реализацию партнёрам, превысившим лимит кредита по договору, как с привязкой к заказу клиента, так и без.
  3. При проведении система сообщает "Внимание, сумма реализаций больше, чем допустимая задолженность по договору на 196 179,15"
  4. Учёт ведётся по выставленным реализациям с учетом текущей.
  5. Даёт спать спокойно, не думая, что кто-то берёт и не отдаёт безлимитно.

Установка расширения

  1. Подсистема "НСИ и администрирование"
  2. Печатные формы, отчеты и обработки
  3. Расширения
  4. Добавить из файла
  5. Снять флажок безопасный режим
